PART 2: UNDERSTANDING THE SYSTEM

🤖 Chapter 5: Meet Your AI Development Team

Think of this like hiring a construction company. You have a project manager (Orchestrator) who coordinates specialists (carpenters, electricians, plumbers). You don't need to manage each specialist directly - you work with the project manager.

🔑 Key Insight

AI agents are incredibly capable but need clear instructions. Think of them as highly skilled contractors who can work 24/7 but need detailed project specifications to deliver exactly what you want.

The Two Types of AI Workers:

🎭 Managers (Orchestrators)
Coordinate the work
Like project managers
🔧 Workers (Specialists)
Do specific tasks
Like contractors

Your AI Team Members:

🎯 Project Manager AI ORCHESTRATOR

What they do: Understand your requirements and coordinate specialists

Business equivalent: General contractor who manages subcontractors

When you need them: Any project with multiple parts

Cost consideration: More expensive per task but saves you management time

🎨 UI/UX Designer AI SPECIALIST

What they do: Create user interfaces and customer experiences

Business equivalent: Web designer who knows what customers expect

When you need them: Anything customer-facing

Example output: "Design a booking form that matches your brand"

🗄️ Database Architect AI SPECIALIST

What they do: Organize and structure your business data

Business equivalent: Filing system designer for digital records

When you need them: Storing customer info, inventory, orders

Example output: "Structure for tracking customer appointments"

🔄 Chapter 6: How They Work Together

Here's the magic: agents don't write code directly. They create detailed plans, then the main AI (Claude) reads all the plans and builds your system. This prevents chaos and ensures everything works together.

The Three-Phase Process:

Phase 1: Research (Understanding Your Needs)

What Happens You say: "I need online booking for my salon" ↓ Manager AI: "I need to understand this business" ↓ Deploys: UI Designer + Database Expert + Calendar Specialist ↓ They research: Your industry, competition, best practices

Phase 2: Planning (Creating Blueprints)

Plans Created UI Designer creates: - Booking form design - Customer journey map - Mobile responsiveness plan Database Expert creates: - Customer data structure - Appointment storage system - Staff availability tracking Calendar Specialist creates: - Scheduling logic - Conflict prevention - Reminder system

Phase 3: Building (Actual Implementation)

Final Build Main AI (Claude) reads all plans and: 1. Creates the booking interface 2. Sets up the database 3. Implements scheduling logic 4. Adds email/SMS reminders 5. Tests everything 6. Delivers working system RESULT: Your custom booking system!

⚠️ Technical Setup Required

This section requires installing software and using command-line tools. If you're not comfortable with this, consider getting help from a tech-savvy friend or hiring someone for the initial setup (usually 1-2 hours).

📋 Before You Start - Prerequisites

  • ✅ Computer with Windows 10+, Mac OS 10.15+, or Linux
  • ✅ Admin access to install software
  • ✅ Internet connection
  • ✅ 2GB free disk space
  • ✅ A text editor (Notepad++ or VS Code recommended)

Step-by-Step Setup Guide:

Step 1: Install Node.js (Required) +
  1. Go to nodejs.org
  2. Click the big green button (LTS version)
  3. Run the downloaded file
  4. Click "Next" through all screens (default options are fine)
  5. Restart your computer

Verify it worked: Open Terminal/Command Prompt and type: node --version

You should see something like: v20.11.0

Step 2: Install MultiAgent Claude +

Open Terminal (Mac) or Command Prompt (Windows):

npm install -g multiagent-claude-cli # If that doesn't work, try: sudo npm install -g multiagent-claude-cli

Common Issues:

  • If "npm: command not found" - Node.js didn't install correctly
  • If "permission denied" - Add 'sudo' before the command (Mac/Linux)
  • If still issues - Try: npx multiagent-claude-cli instead
Step 3: Initialize Your First Project +

1. Create a folder for your project:

# Windows: mkdir C:\MyProjects\my-business-app cd C:\MyProjects\my-business-app # Mac: mkdir ~/Projects/my-business-app cd ~/Projects/my-business-app

2. Initialize MultiAgent Claude:

mac init # The wizard will ask: # - Project type? → Choose your type # - Add memory system? → Yes (recommended) # - Add testing? → Yes for production # - CI/CD? → No for first project

🔧 Chapter 11: Troubleshooting Common Issues

Every business owner runs into these issues. Here's how to solve them:

"MAC command not found" +

Problem: Terminal doesn't recognize 'mac' command

Solutions (try in order):

  1. Close and reopen your terminal/command prompt
  2. Reinstall: npm install -g multiagent-claude-cli
  3. Use npx instead: npx mac init
  4. Check PATH: echo $PATH (should include npm)
"AI creates confusing plans" +

Problem: Plans are too technical or miss the point

Solutions:

  • Be more specific: "for my hair salon" not "for my business"
  • Include examples: "like Calendly but simpler"
  • State constraints: "must work on phones"
  • Ask for explanations: "explain in simple terms"

Better request example:

"Build an appointment booking system for my hair salon. Requirements: - Customers pick stylist, service, date/time - Block lunch hours (12-1pm) - Send SMS reminders - Work on mobile phones Explain each part in simple terms."
"It's not what I wanted" +

Problem: Final result doesn't match expectations

Prevention is key:

  1. Always review plans before building
  2. Start with simple version, then add features
  3. Use examples: "like this website: [URL]"
  4. Iterate: "Change X to be more like Y"

Recovery options:

  • Request specific changes: "Make the form bigger"
  • Start fresh with clearer requirements
  • Use visual examples/mockups if possible
"How do I deploy/launch this?" +

Making your project live:

  1. Simple websites: Use Vercel or Netlify (free)
  2. Business apps: Use Heroku or Railway ($5-20/month)
  3. Internal tools: Run on your computer

Deployment command:

# For Vercel (recommended): npm install -g vercel vercel # Follow prompts - it's automatic!
